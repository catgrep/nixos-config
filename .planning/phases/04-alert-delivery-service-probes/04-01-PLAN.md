<task type="auto">
  <name>Task 2: Configure Grafana SMTP, alerting, and 6 migrated alert rules</name>
  <files>modules/gateway/grafana.nix</files>
  <action>
    Modify `modules/gateway/grafana.nix` to add all alert delivery infrastructure. This is one file with several additions:

    **1. Add SOPS secret declaration for SMTP password:**
    ```nix
    sops.secrets.grafana_smtp_password = {
      owner = "grafana";
      group = "grafana";
      mode = "0400";
    };
    ```
    Follow the exact pattern of the existing `grafana_admin_password` secret.

    **2. Add SMTP settings to `services.grafana.settings`:**
    ```nix
    smtp = {
      enabled = true;
      host = "smtp.gmail.com:587";
      user = "<USER_EMAIL>";  # From checkpoint task 1
      password = "$__file{${config.sops.secrets.grafana_smtp_password.path}}";
      from_address = "<USER_EMAIL>";
      from_name = "Homelab Alerts";
      startTLS_policy = "MandatoryStartTLS";
    };
    ```

    **3. Add stable UID to existing Prometheus datasource provisioning:**
    In `services.grafana.provision.datasources.settings.datasources`, add `uid = "prometheus";` to the existing Prometheus datasource entry. This is critical -- alert rules reference `datasourceUid = "prometheus"` and will break without it.

    **4. Add alerting contact point provisioning:**
    ```nix
    services.grafana.provision.alerting.contactPoints.settings = {
      apiVersion = 1;
      contactPoints = [{
        orgId = 1;
        name = "email-alerts";
        receivers = [{
          uid = "email-alerts-uid";
          type = "email";
          settings = {
            addresses = "<USER_EMAIL>";  # From checkpoint task 1
            singleEmail = false;
          };
          disableResolveMessage = false;
        }];
      }];
    };
    ```

    **5. Add notification policy provisioning:**
    ```nix
    services.grafana.provision.alerting.policies.settings = {
      apiVersion = 1;
      policies = [{
        orgId = 1;
        receiver = "email-alerts";
        group_by = [ "grafana_folder" "alertname" ];
        routes = [{
          receiver = "email-alerts";
          object_matchers = [
            [ "severity" "=~" "critical|warning" ]
          ];
        }];
      }];
    };
    ```

    **6. Add 6 Grafana-managed alert rules mirroring existing Prometheus rules:**
    ```nix
    services.grafana.provision.alerting.rules.settings = {
      apiVersion = 1;
      groups = [{
        orgId = 1;
        name = "homelab_infrastructure";
        folder = "Alerting";
        interval = "1m";
        rules = [ ... ];  # 6 rules below
      }];
    };
    ```

    Each rule follows the three-step data model (Query A -> Reduce B -> Threshold C). Create these 6 rules with unique UIDs:

    1. **host_down** -- `up{job="node-exporter"} == 0` for 5m, severity: critical
    2. **high_disk_usage** -- `(node_filesystem_avail_bytes / node_filesystem_size_bytes) * 100 < 10` for 5m, severity: warning
    3. **high_memory_usage** -- `(node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100 < 10` for 5m, severity: warning
    4. **zfs_pool_unhealthy** -- `node_zfs_zpool_health_state{state!="online"} > 0` for 5m, severity: critical
    5. **high_cpu_temp** -- `node_hwmon_temp_celsius > 80` for 5m, severity: warning
    6. **camera_storage_high** -- `(node_filesystem_avail_bytes{mountpoint="/mnt/cameras"} / node_filesystem_size_bytes{mountpoint="/mnt/cameras"}) * 100 < 20` for 5m, severity: warning

    All rules use `datasourceUid = "prometheus"` for the Query step and `datasourceUid = "__expr__"` for the Reduce and Threshold steps.

    For rules checking "value < threshold" (disk, memory, camera storage): Query returns the percentage, Reduce with `last`, Threshold with `type = "lt"` and appropriate params.
    For rules checking "value > threshold" (temp): Use `type = "gt"`.
    For rules checking "equals 0" (host_down): Query `up{job="node-exporter"}`, Reduce with `last`, Threshold with `type = "lt"` and params = [1] (i.e., fire when value < 1).
    For ZFS unhealthy: Query `node_zfs_zpool_health_state{state!="online"}`, Reduce with `last`, Threshold with `type = "gt"` and params = [0].

    Use annotations matching the existing Prometheus rule summaries.

    **Important:** Do NOT modify prometheus.nix. Keep existing Prometheus ruleFiles as defense-in-depth.

    See the research document Pattern 1, 3, 5 for exact data model structure.
