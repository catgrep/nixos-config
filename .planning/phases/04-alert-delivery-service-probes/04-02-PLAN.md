    **1. Create `modules/gateway/blackbox.nix`:**

    New file with blackbox exporter configuration defining three probe modules:

    ```nix
    # SPDX-License-Identifier: GPL-3.0-or-later

    { config, lib, pkgs, ... }:

    {
      services.prometheus.exporters.blackbox = {
        enable = true;
        configFile = pkgs.writeText "blackbox.yml" (builtins.toJSON {
          modules = {
            http_2xx = {
              prober = "http";
              timeout = "10s";
              http = {
                valid_http_versions = [ "HTTP/1.1" "HTTP/2.0" ];
                preferred_ip_protocol = "ip4";
                follow_redirects = true;
              };
            };
            icmp_ping = {
              prober = "icmp";
              timeout = "5s";
            };
            tls_connect = {
              prober = "http";
              timeout = "10s";
              http = {
                preferred_ip_protocol = "ip4";
                valid_http_versions = [ "HTTP/1.1" "HTTP/2.0" ];
              };
            };
          };
        });
      };
    }
    ```

    Notes:
    - No firewall port needed -- Prometheus scrapes localhost:9115
    - NixOS blackbox module auto-sets CAP_NET_RAW for ICMP
    - `tls_connect` is an HTTP prober (not TCP) -- it connects via HTTPS and exposes `probe_ssl_earliest_cert_expiry`

<task type="auto">
  <name>Task 2: Add 3 probe-based alert rules to Grafana alerting</name>
  <files>modules/gateway/grafana.nix</files>
  <action>
    Add a second alert rule group to the existing `services.grafana.provision.alerting.rules.settings.groups` list in grafana.nix (created in Plan 01).

    Add a new group named `"homelab_probes"` in the same `"Alerting"` folder with `interval = "1m"` containing 3 rules:

    **1. service_down** -- "Service Down (Probe Failed)"
    - PromQL query: `probe_success{job="blackbox-http"}`
    - Reduce: last
    - Threshold: lt, params [1] (fires when probe_success < 1, i.e., == 0)
    - For: 2m
    - Severity: critical
    - Annotation: `"Service {{ $labels.instance }} is unreachable"`

    **2. host_unreachable** -- "Host Unreachable (ICMP Failed)"
    - PromQL query: `probe_success{job="blackbox-icmp"}`
    - Reduce: last
    - Threshold: lt, params [1] (fires when < 1, i.e., probe failed)
    - For: 2m
    - Severity: critical
    - Annotation: `"Host {{ $labels.instance }} is unreachable via ICMP"`

    **3. tls_cert_expiring** -- "TLS Certificate Expiring Soon"
    - PromQL query: `(probe_ssl_earliest_cert_expiry{job="blackbox-tls"} - time()) / 86400`
    - This gives days until expiry.
    - Reduce: last
    - Threshold: lt, params [14] (fires when < 14 days until expiry)
    - For: 1h (avoid transient spikes)
    - Severity: warning
    - Annotation: `"TLS certificate for {{ $labels.instance }} expires in {{ $value }} days"`

    All rules use `datasourceUid = "prometheus"` for the Query step and `datasourceUid = "__expr__"` for Reduce and Threshold steps. Set `noDataState = "NoData"` and `execErrState = "Alerting"`.

    The three-step data model format (Query A -> Reduce B -> Threshold C) follows the exact same pattern as the 6 rules in Plan 01. Reference those for structure.
  </action>
