In `modules/gateway/grafana.nix`, modify the `homelab_infrastructure` rule group:

1. **REMOVE** the existing `high_disk_usage` rule (uid "high_disk_usage", currently at line ~247-304).

2. **ADD** two new graduated disk alert rules in its place:

   **disk_usage_warning** (Warning: >80% used = <20% free):
   - uid = "disk_usage_warning"
   - title = "Disk Usage Warning (>80%)"
   - PromQL expr: `(node_filesystem_avail_bytes{fstype!~"tmpfs|overlay|squashfs|devtmpfs|fuse\\.mergerfs",mountpoint!~"/boot.*|/run.*|/sys.*|/proc.*|/dev.*"} / node_filesystem_size_bytes) * 100`
   - Threshold: refId C with evaluator `{ params = [ 20 ]; type = "lt"; }` (less than 20% free)
   - "for" = "5m", labels = { severity = "warning" }
   - annotation summary = "Disk usage above 80% on {{ $labels.instance }} mount {{ $labels.mountpoint }}"
   - Use the same three-step data model pattern (Query A -> Reduce B -> Threshold C) as existing rules
   - relativeTimeRange = { from = 300; to = 0; }
   - noDataState = "NoData", execErrState = "Alerting", isPaused = false

   **disk_usage_critical** (Critical: >90% used = <10% free):
   - uid = "disk_usage_critical"
   - title = "Disk Usage Critical (>90%)"
   - Same PromQL expr as warning (same filesystem filtering)
   - Threshold: refId C with evaluator `{ params = [ 10 ]; type = "lt"; }` (less than 10% free)
   - "for" = "5m", labels = { severity = "critical" }
   - annotation summary = "Disk usage above 90% on {{ $labels.instance }} mount {{ $labels.mountpoint }}"
   - Same three-step data model, same relativeTimeRange, noDataState, execErrState, isPaused

3. **ADD** a new CPU sustained usage alert rule to the same `homelab_infrastructure` group:

   **high_cpu_usage** (Warning: >90% CPU for 5min):
   - uid = "high_cpu_usage"
   - title = "High CPU Usage (>90% sustained)"
   - PromQL expr: `100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)`
   - Threshold: refId C with evaluator `{ params = [ 90 ]; type = "gt"; }` (greater than 90%)
   - "for" = "5m", labels = { severity = "warning" }
   - annotation summary = "CPU usage above 90% sustained for 5+ minutes on {{ $labels.instance }}"
   - relativeTimeRange = { from = 600; to = 0; } (10 min window for rate calculation)
   - Same three-step data model, noDataState = "NoData", execErrState = "Alerting", isPaused = false

4. **Update the existing `camera_storage_high` rule** (uid "camera_storage_high") to add the same fstype filter for consistency: add `fstype!~"tmpfs|overlay|squashfs|devtmpfs|fuse\\.mergerfs"` to its expr. This is a minor consistency improvement -- the mountpoint filter already limits it to /mnt/cameras, but adding fstype filter is belt-and-suspenders.

In `modules/gateway/prometheus.nix`, update the `ruleFiles` section:

5. **REPLACE** the existing `HighDiskUsage` rule with two graduated rules:

   **HighDiskUsageWarning:**
   ```yaml
   - alert: HighDiskUsageWarning
     expr: (node_filesystem_avail_bytes{fstype!~"tmpfs|overlay|squashfs|devtmpfs|fuse.mergerfs",mountpoint!~"/boot.*|/run.*|/sys.*|/proc.*|/dev.*"} / node_filesystem_size_bytes) * 100 < 20
     for: 5m
     labels:
       severity: warning
     annotations:
       summary: "Disk usage above 80% on {{ $labels.instance }} mount {{ $labels.mountpoint }}"
   ```

   **HighDiskUsageCritical:**
   ```yaml
   - alert: HighDiskUsageCritical
     expr: (node_filesystem_avail_bytes{fstype!~"tmpfs|overlay|squashfs|devtmpfs|fuse.mergerfs",mountpoint!~"/boot.*|/run.*|/sys.*|/proc.*|/dev.*"} / node_filesystem_size_bytes) * 100 < 10
     for: 5m
     labels:
       severity: critical
     annotations:
       summary: "Disk usage above 90% on {{ $labels.instance }} mount {{ $labels.mountpoint }}"
   ```

6. **ADD** the CPU sustained usage rule to Prometheus ruleFiles:

   ```yaml
   - alert: HighCPUSustained
     expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 90
     for: 5m
     labels:
       severity: warning
     annotations:
       summary: "CPU usage above 90% sustained for 5+ minutes on {{ $labels.instance }}"
   ```

Important: Use `rate()` not `irate()` -- rate() gives a smoother signal suitable for alerting. The PromQL escaping in Nix multi-line strings ('' '') does not need backslash escaping for double quotes -- use `{mode="idle"}` directly inside `'' ''`.
