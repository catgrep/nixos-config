<task type="auto">
  <name>Task 1: Add Frigate notification automation to home-assistant.nix</name>
  <files>modules/automation/home-assistant.nix</files>
  <action>
Add a single automation entry to the existing `"automation manual" = [ ];` list in `modules/automation/home-assistant.nix`. The automation should:

1. **Alias:** "Frigate Alert Notification"
2. **Description:** "Send push notification with snapshot when Frigate detects person, car, or package"
3. **Mode:** "parallel" with max 10 (multiple cameras can fire simultaneously)
4. **Trigger:** MQTT trigger on topic `frigate/reviews` with payload `alert` and value_template `{{ value_json['after']['severity'] }}`
5. **Conditions (both must pass):**
   - Object type filter: `{{ trigger.payload_json['after']['data']['objects'] | select('in', ['person', 'car', 'package']) | list | length > 0 }}`
   - Detections array not empty (defensive): `{{ trigger.payload_json['after']['data']['detections'] | length > 0 }}`
6. **Action:** Call `notify.mobile_app_PLACEHOLDER_DEVICE_ID` (use literal string `PLACEHOLDER_DEVICE_ID` -- will be replaced after Companion app setup in Task 2) with:
   - `title`: `{{ trigger.payload_json['after']['data']['objects'] | sort | join(', ') | title }} Detected`
   - `message`: `{{ trigger.payload_json['after']['camera'] | replace('_', ' ') | title }}`
   - `data.image`: `/api/frigate/notifications/{{ trigger.payload_json['after']['data']['detections'][0] }}/snapshot.jpg`
   - `data.tag`: `{{ trigger.payload_json['after']['id'] }}` (review ID for deduplication)
   - `data.when`: `{{ trigger.payload_json['after']['start_time'] | int }}`
   - `data.entity_id`: `camera.{{ trigger.payload_json['after']['camera'] }}`
   - `data.url`: `/api/frigate/notifications/{{ trigger.payload_json['after']['data']['detections'][0] }}/clip.mp4` (iOS tap action)
   - `data.clickAction`: same as url (Android tap action)
   - `data.group`: `frigate-{{ trigger.payload_json['after']['camera'] }}` (group by camera)
   - `data.actions`: one URI action button "View Clip" pointing to the clip URL

Use the complete code example from the research as reference (see 02-RESEARCH.md "Complete Notification Automation in Nix" section). All Jinja2 `{{ }}` templates are plain Nix strings -- no escaping needed since Nix uses `${ }` for interpolation.

After writing the code, run `make build-ser8` to verify the Nix configuration compiles without errors. Do NOT deploy yet.

IMPORTANT: Use `detections[0]` (detection event ID) for snapshot/clip URLs, NOT `after.id` (review ID). The review ID is only for the notification `tag`. Using review ID for snapshots returns 404.
