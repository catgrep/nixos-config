**Part A: Create dashboards/uptime.json**

Create a new Grafana dashboard JSON file at `dashboards/uptime.json` with the following structure:

Dashboard metadata:
- title: "Service Uptime & Status"
- uid: "uptime-status" (must be unique across all dashboards)
- timezone: "browser"
- refresh: "1m"
- time range: last 24 hours (from: "now-24h", to: "now")

Row 1 -- "Service Availability (24h)" -- Stat panels:
Create one stat panel per HTTP-probed service showing 24h availability percentage. Use `avg_over_time(probe_success{job="blackbox-http",instance="INSTANCE_URL"}[24h]) * 100` for each. Services and their instance labels (from prometheus.nix blackbox-http targets):
- Jellyfin: `http://192.168.68.65:8096`
- Sonarr: `http://192.168.68.65:8989`
- Radarr: `http://192.168.68.65:7878`
- Prowlarr: `http://192.168.68.65:9696`
- qBittorrent: `http://192.168.68.65:8080`
- SABnzbd: `http://192.168.68.65:8085`
- Frigate: `http://192.168.68.65:80`
- Home Assistant: `http://192.168.68.65:8123`

Stat panel config for each:
- type: "stat"
- Unit: "percent" (short)
- Thresholds: green >= 99, yellow >= 95, red < 95
- Panel title: friendly service name (e.g., "Jellyfin", "Sonarr", etc.)
- Decimals: 2
- Color mode: "background"
- Grid position: arrange 4 per row (w=6 each in 24-col grid)

Row 2 -- "HTTP Service Status" -- State-timeline panel:
- type: "state-timeline"
- Query: `probe_success{job="blackbox-http"}`
- datasource uid: "prometheus"
- Value mappings: 0 = "DOWN" (red), 1 = "UP" (green)
- Use Grafana field overrides (displayName) to map each instance URL to a friendly name:
  - `http://192.168.68.65:8096` -> "Jellyfin"
  - `http://192.168.68.65:8989` -> "Sonarr"
  - `http://192.168.68.65:7878` -> "Radarr"
  - `http://192.168.68.65:9696` -> "Prowlarr"
  - `http://192.168.68.65:8080` -> "qBittorrent"
  - `http://192.168.68.65:8085` -> "SABnzbd"
  - `http://192.168.68.65:80` -> "Frigate"
  - `http://192.168.68.65:8123` -> "Home Assistant"
- Panel height: 8 grid units (h=8)
- Panel width: full width (w=24)
- Legend: show values
- Tooltip: all series

Row 3 -- "Host Reachability" -- State-timeline panel:
- type: "state-timeline"
- Query: `probe_success{job="blackbox-icmp"}`
- datasource uid: "prometheus"
- Same value mappings as Row 2
- Field overrides for friendly names:
  - `192.168.68.65` -> "ser8"

  - `192.168.68.56` -> "pi4"
- Panel height: 6 grid units
- Panel width: full width

- Thresholds: green > 14, yellow > 7, red <= 7
- Unit: "days" (short: "d")
- Decimals: 0
- Color mode: "background"

- Grid position: 3 per row (w=8 each)

For all panels, use datasource: `{ "type": "prometheus", "uid": "prometheus" }`.

**Part C: Configure msmtp and ZFS zed on ser8** (skip if user said "skip-zed")

In `hosts/ser8/configuration.nix`:

1. Add SOPS secret declaration:
   ```nix
   sops.secrets.gmail_smtp_password = {
     mode = "0400";
   };
   ```

2. Add msmtp as lightweight MTA:
   ```nix
   programs.msmtp = {
     enable = true;
     setSendmail = true;  # Makes msmtp the system sendmail
     defaults = {
       auth = true;
       tls = true;
       tls_starttls = true;
       logfile = "/var/log/msmtp.log";
     };
     accounts.default = {
       host = "smtp.gmail.com";
       port = 587;
       from = "shadbangus@gmail.com";
       user = "shadbangus@gmail.com";
       passwordeval = "cat ${config.sops.secrets.gmail_smtp_password.path}";
     };
   };
   ```

3. Add ZFS Event Daemon configuration:
   ```nix
   services.zfs.zed = {
     enableMail = true;
     settings = {
       ZED_EMAIL_ADDR = [ "catgrep@sudomail.com" ];
       ZED_NOTIFY_VERBOSE = true;
       ZED_SCRUB_AFTER_RESILVER = true;
     };
   };
   ```

4. Ensure `/var/log/msmtp.log` persists (ser8 uses impermanence). Check if `/var/log` is already in the impermanence config. If not, the log file is non-critical and will recreate on boot -- no action needed. The important thing is that msmtp functions; the log is just for debugging.

Important: Use `config.sops.secrets.gmail_smtp_password.path` for the passwordeval -- this is the standard SOPS-NixOS pattern to reference the decrypted secret path at runtime.
