User performs physical verification of the notification pipeline by triggering real Frigate detections and confirming push notifications arrive on the phone with correct content.
  </action>
  <what-built>
The Frigate notification automation is deployed and active in HA. It should send push notifications with snapshot images when Frigate detects a person, car, or package on any of the 3 cameras (driveway, front_door, garage). Notifications use tag-based deduplication and include a tap action to view the clip.
  </what-built>
  <how-to-verify>
**Step 1: Test basic notification delivery (manual trigger)**
1. Open HA web UI -> Developer Tools -> Services
2. Select `notify.mobile_app_<your_device_id>`
3. Enter this YAML:
```yaml
title: "Test Notification"
message: "Frigate notification pipeline test"
data:
  tag: "test-manual"
```
4. Click "Call Service"
5. Confirm: Phone receives the test notification

**Step 2: Test with real Frigate detection**
1. Walk in front of one of the cameras (driveway, front_door, or garage) to trigger a person detection
2. Wait up to 30 seconds
3. Confirm: Phone receives a push notification with:
   - Title containing "Person Detected"
   - Message showing the camera name (e.g., "Front Door")
   - A snapshot image showing the detected person

**Step 3: Verify deduplication**
1. Stay in camera view for 10-20 seconds (Frigate will send review updates)
2. Confirm: The notification updates IN-PLACE (no duplicate notifications for the same event)
3. Check notification count -- should be 1, not growing

**Step 4: Verify tap action**
1. Tap the notification
2. Confirm: It opens the clip or event detail (may show partial clip if event is still in progress, or may redirect to HA -- both are acceptable)

**Step 5: Test car detection (if possible)**
1. If a car drives past a camera, confirm a "Car Detected" notification arrives
2. If not easily triggerable, skip -- the automation uses the same code path for all object types

**If notifications are NOT arriving:**
- Check HA Settings -> Automations -> "Frigate Alert Notification" -> click the 3 dots -> Traces -> look for recent traces
- If no traces: Frigate is not publishing alerts. Check Frigate UI for recent events.
- If traces show errors: report the error message
- Check HA logs: `ssh bdhill@ser8 'sudo journalctl -u home-assistant -n 50 --no-pager | grep -iE "(error|notify|automation)"'`
  </how-to-verify>
  <resume-signal>Report results: (1) Did test notification arrive? (2) Did real detection notification arrive with snapshot? (3) Did dedup work? (4) Did tap action work? Describe any issues.</resume-signal>
